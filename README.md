# React для Vue-разработчиков

Курс-мост для тех, кто уже писал на Vue, что-то успел потрогать в React — и застрял. Это не курс «React с нуля»: здесь не будет глав про то, что такое компонент или зачем нужен пропс. Здесь разбирается ровно то место, где ваша Vue-интуиция начинает врать вам про React, — тема за темой.

## Завязка

Станция «Stellar Burger Nine» много лет держала автоматизированные бургерные по всему сектору на движке Proxy Drive — реактивном ядре, которое само отслеживало, что изменилось, и само перекрашивало нужные панели. Вы были инженером именно на этом движке, в колонии Vue Nebula.

Флот-командование решило унифицировать все терминалы нового поколения на другом ядре — React Core: без автослежения, зато предсказуемо и с огромным пулом инженеров, которых можно нанять в любом секторе. Станцию Nine пересобирают первой. Вас перевели туда не потому, что вы не умеете строить интерфейсы — умеете, и хорошо, — а потому что вам нужно быстро переучиться *думать* по-другому. Экипаж станции не будет объяснять вам, что такое состояние или компонент. Они будут объяснять, чем их подход отличается от того, к чему вы привыкли на Proxy Drive.

Каждый модуль курса — это смена на станции. К концу смены вы допиливаете очередной кусок нового заказ-терминала Stellar Burger.

## Кому подойдёт курс

- Вы писали на Vue (Options API и/или Composition API) и уверенно ориентируетесь в SPA-разработке.
- Вы уже проходили React (курс, туториал, рабочий проект) — но большинство тем всё ещё путаются или ощущаются «не как должны».
- Вам не нужно объяснять, что такое JS-модуль, npm, git или HTTP-запрос.

## Как устроен курс

- **Самостоятельный темп.** Единого расписания нет — идёте по модулям в своём ритме.
- **Практика через код, а не только чтение.** Каждый модуль (кроме нулевого) — это папка с упражнением, эталонным решением и падающими тестами, которые становятся проходящими, когда упражнение решено верно.
- **TypeScript — с первого модуля, а не отдельным блоком в конце.** В каждой теме сначала разбирается идея на обычном React (JS/JSX), а в конце темы появляется **инспектор Тай** (Ty, от *Type*) — придирчивый инспектор соответствия Флота — и показывает, как то же самое написать с типами. Так вы сразу видите, *зачем* нужен тип, а не зубрите синтаксис в отрыве от задачи.
- **Основной сюжетный проект — клон Stellar Burger.** Через все модули 01–10 сквозным примером идёт конструктор космобургеров: список ингредиентов, сборка заказа, авторизация, роутинг, лента заказов по WebSocket. Это осознанно тот же домен, что и в промышленном приложении курса — вы уже понимаете предметную область, поэтому вся энергия уходит на React, а не на придумывание что вообще строить.
- **Дополнительные примеры (`complementary/`) — на других доменах.** Чтобы не спутать «я понял React» с «я запомнил именно этот конструктор бургеров», в каждом модуле есть маленький изолированный пример на другую тему (счётчик, тема оформления, форма обратной связи и т.д.), где та же идея видна в чистом виде.
- **Самопроверка через тесты, не через ревью.** Курс самостоятельный, поэтому обратная связь — это автотесты, а не человек. В каждом модуле тесты проверяют только то, что добавлено в этом модуле (`exercise/`), плюс лёгкий smoke-тест на то, что накопленное Stellar Burger приложение в принципе не развалилось.
- **Входная диагностика + мини-квиз в конце каждого модуля.** Перед модулем 01 — короткая диагностика ([assessment](assessment)): 10 вопросов на типичные Vue-рефлексы, которые ломаются в React, каждый привязан к модулю. Она не блокирует прохождение — она подсказывает, что можно пробежать по диагонали, а что проходить полностью. Каждый модуль 00–09 заканчивается разделом «Проверь себя» в README — несколько вопросов с разбором каждого варианта ответа (не только правильного), ответы спрятаны под `<details>`.

## Формат модуля (01–10)

```
NN-topic-name/
  README.md       теория + сюжет смены + «Проверь себя» в конце + что нужно сделать
  exercise/        стартовый код Stellar Burger с TODO и падающими тестами
  solution/         эталонное решение
  complementary/    маленький изолированный пример на другую тему
```

`exercise/` каждого модуля стартует с уже готового решения предыдущего модуля — если что-то не получилось в прошлый раз, это не блокирует переход дальше.

## Как посмотреть модуль в браузере

Репозиторий — единый npm workspace, поэтому `npm install` нужно один раз, из корня. Дальше любой проект (`exercise`/`solution`/`complementary` любого модуля) можно поднять без `cd`, указав его имя пакета флагом `-w`:

```sh
npm install                                            # один раз, из корня
npm run dev -w 01-components-jsx-solution                # http://localhost:5173
npm run dev -w 02-state-and-reactivity-exercise -- --port 5121  # свой порт, если нужно поднять несколько сразу
```

Имя пакета = `NN-slug-variant` (например, `02-state-and-reactivity-solution`) — совпадает с полем `"name"` в `package.json` каждого проекта.

Для тех, кто открывает этот репозиторий как отдельный проект в Claude Code: в [.claude/launch.json](.claude/launch.json) уже настроены именованные конфигурации для текущих модулей (`01-exercise`, `01-solution`, `02-solution` и т.д.), каждая на своём порту — можно поднимать любой из них по имени через `preview_start`, не разбираясь в командах выше.

## Что нужно знать до старта

- JavaScript (ES6+): деструктуризация, замыкания, промисы/async-await, модули.
- Vue (Options API или Composition API) на уровне рабочего проекта.
- Базовый git и командная строка.

## Экипаж станции

- **Капитан Рут (Root)** — командир станции, отвечает за единственную точку входа, из которой растёт всё остальное.
- **Старший инженер Хукс (Hooks)** — заведует состоянием станции, ничего не трогает напрямую, всё — только через официальный канал.
- **Лейтенант-коммандер Эффект (Effect)** — отвечает за связь с внешним миром: сеть, датчики, доки. Требует процедуру очистки перед сменой.
- **Лейтенант Композ (Compose)** — принципиально не пишет одну и ту же процедуру дважды.
- **QM-9 «Редакс» (Redux)** — дроид-квартирмейстер, ведёт единственный официальный журнал склада; менять его напрямую нельзя, только через заявку.
- **Штурман Роутер (Router)** — прокладывает курс между картами станции, включая фокус «показать соседнюю луну в боковом окне, не покидая орбиты».
- **Энсин Форм (Form)** — принимает заявки и анкеты экипажа, ничего не пропускает без валидации.
- **Инспектор Тай (Ty)** — инспектор соответствия Флота, в конце каждой смены проверяет, что ничего не ушло без типов.
- **Старшина Рендер (Render)** — засекает время каждой операции и вылавливает тех, кто переделывает уже сделанную работу.
- **Энсин Реактива (Reactiva)**, осталась на Vue Nebula — иногда выходит на связь, чтобы напомнить, как то же самое делалось на Proxy Drive.

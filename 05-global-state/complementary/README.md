# Дополнительный пример: тема через Context

Тот же принцип «глобального состояния», что и Redux в основном упражнении, но без стора, экшенов и reducer — Context для одного простого флага обычно достаточно сам по себе. Уже решено, TODO нет.

```sh
npm install   # один раз, из корня репозитория (npm workspaces)
npm run dev    # http://localhost:5173
```

`StatusCard` и `ThemeToggle` — соседние компоненты, ни один не является родителем другого. `StatusCard` тем не менее знает текущую тему, ничего не получая пропом — оба читают её из одного `ThemeContext` через `useTheme()`.

Когда браться за Context, а когда за Redux — вопрос масштаба: одно простое значение (тема, язык интерфейса) обычно тянет Context, а состояние с историей изменений, множеством связанных действий и потребностью в отладке (Redux DevTools, time-travel) — уже повод присмотреться к Redux Toolkit, как в основном упражнении этого модуля.
